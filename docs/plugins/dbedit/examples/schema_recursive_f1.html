Recursive JSON Schema Editor

This example is JSON schema editor creates a JSON schema, that defines the valid structure of JSON. Furthermore is demonstrates the many ways you can use recursive schemas (aka self-referential or circular schemas).

  • This JSON editor uses a JSON schema to create a JSON schema. This could be confusing because JSON schema "s" is used to define the structure of a valid JSON "j". A JSON schema "s" is a JSON file itself, so the valid structure "s" can be defined by a JSON schema "s4s".
  • REMARK: For layout purpose in the JSON editor the definition of "object" in the JSON is implemented as an array. This implies that JSON schema must be post-processed after using the JSON editor of Jeremy Dorn.
  • Within array items as long as minLength is 0. See "coworkers" below.
  • In non-default properties. See "mother" below (click the "object properties" button and check "mother")
  • In oneOf as long as it's not the 1st choice. See "bestFriend" below.
  • In patternProperties. Try adding the property "cousin_1" using the "object properties" button.
